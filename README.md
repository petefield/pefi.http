# pefi.http

A C# [Roslyn source generator](https://learn.microsoft.com/en-us/dotnet/csharp/roslyn-sdk/source-generators-overview) that automatically generates strongly-typed `HttpClient` wrappers from [OpenAPI](https://www.openapis.org/) (Swagger) specifications at compile time â€” no runtime reflection, no code-gen scripts, just add an attribute.

## Features

- Generates a complete `HttpClient` wrapper at compile time from an OpenAPI 3.x JSON specification
- Generates C# model classes for all schemas defined in `components/schemas`
- Supports path, query, and header parameters
- Supports JSON request bodies and JSON response deserialization
- Handles all standard HTTP methods: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `HEAD`, `TRACE`
- Zero runtime overhead â€” all code is generated during compilation

## Installation

The package is published to [GitHub Packages](https://github.com/petefield/pefi.http/packages).

Add the GitHub Packages NuGet source to your `nuget.config` (or global NuGet configuration):

```xml
<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <packageSources>
    <add key="github" value="https://nuget.pkg.github.com/petefield/index.json" />
  </packageSources>
</configuration>
```

Then add the package reference to your project:

```xml
<PackageReference Include="pefi.http" Version="*-ci-*" />
```

> **Note:** You need a GitHub Personal Access Token (PAT) with `read:packages` scope to authenticate with GitHub Packages. See [GitHub's documentation](https://docs.github.com/en/packages/working-with-a-github-packages-registry/working-with-the-nuget-registry#authenticating-to-github-packages) for details.

## Usage

### 1. Add your OpenAPI specification to the project

Add your OpenAPI JSON spec file to the project and mark it as an `AdditionalFiles` item:

```xml
<ItemGroup>
  <AdditionalFiles Include="my-api-spec.json" />
</ItemGroup>
```

### 2. Decorate a partial class with the `[GenerateHttpClient]` attribute

```csharp
using pefi.http;

namespace MyApp;

[GenerateHttpClient("my-api-spec.json")]
public partial class MyApiClient
{
}
```

The attribute argument must match the **file name** (not the full path) of the `AdditionalFiles` entry.

### 3. Register and use the generated client

The generated `partial class` adds a constructor that accepts an `HttpClient` and auto-generated async methods for every operation in the spec. Register it like any other typed `HttpClient`:

```csharp
// Program.cs / Startup.cs
builder.Services.AddHttpClient<MyApiClient>(client =>
{
    client.BaseAddress = new Uri("https://api.example.com");
});
```

Then inject and call it:

```csharp
public class MyService
{
    private readonly MyApiClient _client;

    public MyService(MyApiClient client) => _client = client;

    public async Task DoSomethingAsync(CancellationToken ct)
    {
        var results = await _client.GetWeatherForecastAsync(cancellationToken: ct);
    }
}
```

## Generated Code

Given an OpenAPI operation like:

```json
{
  "paths": {
    "/weather": {
      "get": {
        "operationId": "GetWeatherForecast",
        "responses": {
          "200": {
            "content": {
              "application/json": {
                "schema": {
                  "type": "array",
                  "items": { "$ref": "#/components/schemas/WeatherForecast" }
                }
              }
            }
          }
        }
      }
    }
  }
}
```

The generator produces:

```csharp
// <auto-generated/>
public partial class MyApiClient
{
    private readonly HttpClient _httpClient;
    public JsonSerializerOptions _jsonOptions { get; set; }

    public MyApiClient(HttpClient httpClient)
    {
        _httpClient = httpClient ?? throw new ArgumentNullException(nameof(httpClient));
        _jsonOptions = new JsonSerializerOptions { PropertyNameCaseInsensitive = true };
    }

    public async Task<IReadOnlyList<WeatherForecast>?> GetWeatherForecastAsync(
        CancellationToken cancellationToken = default)
    {
        // ... request/response handling
    }
}
```

## Type Mapping

OpenAPI types are mapped to C# types as follows:

| OpenAPI type | Format      | C# type         |
|-------------|-------------|-----------------|
| `string`    | â€”           | `string`        |
| `string`    | `date-time` | `DateTimeOffset` |
| `string`    | `date`      | `DateOnly`      |
| `string`    | `time`      | `TimeOnly`      |
| `string`    | `byte`      | `byte[]`        |
| `string`    | `binary`    | `Stream`        |
| `integer`   | `int32`     | `int`           |
| `integer`   | `int64`     | `long`          |
| `number`    | `float`     | `float`         |
| `number`    | `double`    | `double`        |
| `number`    | â€”           | `decimal`       |
| `boolean`   | â€”           | `bool`          |
| `array`     | â€”           | `IReadOnlyList<T>` |
| `object`    | â€”           | `object`        |

Nullable OpenAPI types (those with `nullable: true` or `| null`) are mapped to their nullable C# equivalents (e.g. `string?`, `int?`).
